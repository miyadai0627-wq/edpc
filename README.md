# Custom Voice Check (カスタム・ボイス・チェック)

## はじめに
本リポジトリは、開発者自身が作ったオリジナル英語学習アプリケーションです。

（※ご利用いただくことでのトラブル等には一切責任を負いかねますのでご了承ください。）

## 概要
自分だけの英語フレーズを登録し、リスニングと発音の練習ができるブラウザ完結型のWebアプリケーションです。
外部のデータベースや有料APIに依存せず、ブラウザの標準機能（Web Speech API, LocalStorage）のみを活用して、単語帳の管理から音声合成（お手本再生）、音声認識（発音チェック）までを行うことができます。

## 💡 背景とコンセプト (Background & Concept)

**開発のきっかけ：自身の英語学習における「リアルな課題」**

本アプリは、開発者自身の英語学習における「インプットとアウトプットのギャップ」を埋めるために誕生しました。文法や単語の基礎知識は既存の学習アプリでインプットできるものの、「実際にフレーズを口に出して定着させる」ための反復練習が圧倒的に不足していると感じていました。
流行りのAI英会話ツールも試しましたが、会話のテンポに流されてしまい、「特定のフレーズを自分が納得いくまで何度も反復練習する」「発音の正確さだけをシンプルかつ厳密に判定してもらう」というストイックな用途には不向きでした。

**コンセプト：徹底反復！自分専用フレーズ・スパーリングアプリ**

そこで、「自分が本当に使いたいフレーズ」だけを登録し、AIの相槌や余計なUIに邪魔されることなく、ひたすら「お手本を聞く → 発音する → 判定する」を繰り返せるスピーキング特化型の反復練習ツールを開発しました。

**【ターゲット層】**
* 基礎学習は終えており、スピーキングの「瞬発力」を鍛えたい人
* 日々忙しく働く中で、通勤時間や寝る前などの限られたスキマ時間を使って、効率よくアウトプットのトレーニングをしたい人
* AI英会話での実践練習の前に、まずは決まったフレーズを自信を持って言えるようになりたい人

**【UI/UXへのこだわり】**
「手軽に、とにかく回数をこなす」ことに特化するため、無駄を徹底的に排除しました。ログインや初期設定の手間をなくし、ブラウザを開いてすぐに練習を開始できる設計にしています。また、AI英会話特有の「会話のテンポ待ち」を解消するため、ワンタップでお手本再生・自分の発音判定が完結する、直感的で軽量なユーザー体験を実現しました。

## ✨ 主な機能 (Features)

1. **Myフレーズ帳 (CRUD機能):** 練習したい英単語やフレーズを日本語訳と共に入力し、ブラウザ上に保存・一覧表示・削除ができます。初回アクセス時にはすぐに練習を始められるよう「初期フレーズ100選」がロードされます。
2. **お手本リスニング (Text-to-Speech):** 登録した英語フレーズをネイティブな発音で読み上げます。練習画面でも何度でも聞き直すことが可能です。
3. **発音チェッカー (Speech-to-Text):** マイクを使って自分が発音した音声をテキスト化し、正解のフレーズと一致しているかを判定します。
4. **【独自機能】難易度切り替え（ゆるめ / 厳しめ）:** ユーザーのレベルに合わせて判定の厳しさをトグルボタンで切り替えることができます。
5. **【独自機能】おしい！(Almost!) 判定:** 完全に一致しなくても、単語の合致率を内部でパーセント計算し、50%以上合っていれば「Almost!」とフィードバックを返す優しい判定システムを搭載しています。

## 🛠 使用技術 (Tech Stack)

* **フロントエンド:** HTML, CSS, JavaScript (Vanilla JS)
* **データ保存:** LocalStorage
* **音声処理:** Web Speech API (SpeechSynthesis / SpeechRecognition)

**【開発ポイント】**
フロントエンドの基礎力を証明するため、あえてReactなどのフレームワークや外部の有料APIを使わず、Vanilla JS（素のJavaScript）とブラウザ標準機能のみで実装しました。配列操作やDOM操作、非同期処理を駆使し、ユーザーのデバイス（マイク・スピーカー）と連動したインタラクティブな機能を実現しています。

## 🔥 開発において苦労した点・こだわった点

### 1. コンピューターの「厳密さ」と戦いから生まれた「Almost!機能」
発音判定のロジックを実装した際、最大の壁となったのが「文字列比較の罠」でした。マイクが聞き取った `are you ready` と、正解データの `Are you ready?`。人間にとっては同じでも、プログラムの厳密な比較（`===`）では「大文字小文字の違い」「ピリオドやクエスチョンマークの有無」によって不正解（Try again!）と弾かれてしまい、非常にストレスを感じる仕様になってしまいました。
そこで、記号を排除し小文字に統一する正規表現（`.toLowerCase().replace(/[.,!?]/g, "").trim()`）を用いて「ゆるめ判定」を実装。さらに、「theが抜けただけで不正解になるのは辛い」という自身の気づきから、文章を単語単位（配列）に分割して合致率を算出し、50%以上なら「おしい！(Almost!)」と返すアルゴリズムを自作しました。エラーから逃げず、ユーザー体験（UX）の向上へと昇華させたこだわりの機能です。

### 2. ブラウザ依存（クロスブラウザ）の壁とトラブルシューティング
実装中、コードは完璧なはずなのに特定環境（Braveブラウザ等）でマイク機能が一切動かないという事象に直面しました。原因を調査した結果、Web Speech APIの裏側で動いているGoogleの音声認識サーバーへの通信が、プライバシー保護機能の強いブラウザではAPI制限・ブロックの対象になるという「ブラウザの仕様の違い」を突き止めました。
開発者ツール（コンソール）のログを読み解き、「コードのバグ」ではなく「実行環境のセキュリティ仕様」であると切り分けられたことは、エンジニアとしての大きな成長と自信に繋がりました。

### 3. 「自分が使うなら」を追求したUX改善
開発途中で実際にアプリをテストプレイした際、「発音テストで間違えた後、もう一度お手本を聞きたいのに、一覧画面に戻らないと聞けない」という不便さに気づきました。そこで急遽、練習画面の問題文の横にもスピーカーボタン（🔊）を配置し、`practiceList[currentIndex]` の状態（State）を正しく参照することで、その場で何度でも発音を確認できる「最強の反復練習ループ」を構築しました。

## 📖 使用方法 (Usage)

本アプリはブラウザ上で完結するため、インストール不要ですぐにご利用いただけます。
※マイク機能を使用するため、**Google Chrome** または **Safari** ブラウザでの利用を推奨します（Brave等のプライバシー強化ブラウザでは音声認識が動作しない場合があります）。

1. **フレーズの追加:** 画面上部の入力欄に英語と日本語を入力し、「＋追加」ボタン（またはEnterキー）を押してMyフレーズ一覧に登録します。
2. **練習の開始:** 「▶ 練習スタート」ボタンを押すと、登録されたフレーズがシャッフルされ、練習画面に切り替わります。
3. **発音のテスト:** * 画面に表示された英語を読み上げます。
   * 「🎤 発音をテスト」ボタンを押し、マイクに向かって発音します。（※初回はマイクの使用許可を求められます）
   * 発音が認識されると、画面に聞き取ったテキストと判定結果（Excellent!! / Almost! / Try again!）が表示されます。
   * 🔊ボタンを押すと、お手本の音声を確認できます。
4. **難易度の変更:** マイクボタン上部の「判定：ゆるめ / 厳しめ」ボタンをタップすると、判定の厳しさを切り替えることができます。
5. **次の問題へ:** 「答えを表示」で日本語訳を確認し、「次の問題へ」で学習を進めます。
